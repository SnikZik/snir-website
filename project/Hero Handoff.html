0. Protected text zones (HARD RULE — read first)

Inviolable rule: No visual layer (knight, rings, BUILD word, glow) may enter the headline zone. Any element that crosses the boundary, even by 1px, is a failed implementation.

  • Rings: bounding box must sit entirely inside the decoration zone.

  • Glow: peak alpha (≥0.20) must sit entirely inside the decoration zone; soft falloff (alpha < 0.10) may bleed but must not affect headline contrast.

Headline line-break rule (DETERMINISTIC — replaces the previous max-width approach).
Do not rely on natural wrapping or max-width for line breaks. Use explicit per-word markup so the result cannot change across browsers, fonts, or fallbacks.
  1. Each word lives in its own <span class="word">…</span>.
  2. The headline has two visual lines on desktop, four on mobile. Toggle each word's display via a data-break attribute and CSS, not <br>.
  3. Apply white-space: nowrap on each word to guarantee no intra-word break.
Markup (use exactly): 
<h1 class="hero__title">
  <span class="word">Understand</span> <span class="word w-deeply">deeply<span class="dot"></span></span>
  <span class="word w-build">Build</span> <span class="word w-fast">fast<span class="dot"></span></span>
</h1>
Rules: 
.hero__title{display:flex;flex-wrap:wrap;column-gap:.22em;row-gap:0;}
.hero__title .word{white-space:nowrap;display:inline-block;}

/* DESKTOP — 2 lines: "Understand deeply." / "Build fast." */
.hero__title .w-build{flex-basis:100%;} /* forces line break before "Build" */

/* MOBILE — 4 lines: Understand / deeply. / Build / fast. */
@media (max-width:768px){
  .hero__title .word{flex-basis:100%;} /* every word on its own line */
}
Result: line breaks are forced by flex flex-basis:100%, not by font metrics. Cannot drift if the font fails to load or the user zooms.

7. CSS architecture

  1. One stylesheet for the hero: /styles/hero.css. No utility CSS, no Tailwind classes, no inline styles.
  2. All shared values are CSS custom properties on :root (see §3).
  3. The hero is positioned via a single position: relative on .hero; every layer inside is position: absolute except .hero__copy and .hero__topbar.
  4. .hero__topbar is position: absolute; top:0; left:0; right:0; — not fixed, not sticky.
  5. .hero__copy is position: relative with explicit z-index; it does not float or grid.
  6. No transforms on any layer except the translateY(-2px) on dot squares to optically center them.
  7. No animation. No hover state changes that move layout. Hover on links is a single color change only.
  8. One media query break: @media (max-width: 768px). Mobile values fully override desktop within it.

10. Responsive rules

  1. Single breakpoint: @media (max-width: 768px). Below it, all mobile values from §9 apply. Above it, all desktop values from §8 apply.
  2. No fluid clamp() typography. Sizes are fixed within each breakpoint.
  3. Between 768 and 1440, the hero uses the desktop spec but allows horizontal padding to scale: padding-inline: clamp(20px, 4vw, 56px). Headline and decorative element sizes do not change.
  4. Above 1440, the hero is centered in a max-width:1440px container; outer paper background fills the rest of the viewport.
  5. Hide the desktop nav and CTA below 768. Show the hamburger above 0 and below 768. The hamburger is hidden ≥768.
  6. The headline switches from 2 lines (desktop) to 4 lines (mobile) by container max-width alone — no <br>, no JS.

12. Delete from current implementation

  • Any linear-gradient on body or .hero backgrounds. Background must be flat #F2EFEA.
  • Any backdrop-filter, mix-blend-mode, or box-shadow on the glow, rings, knight, or BUILD word.
  • Any <br> tags inside the headline. Replace with <span class="line"> wrappers + max-width.
  • Any framework grid wrapper (.container, .row, .col-*) inside the hero. The hero positions its own children.
  • Any animation, transition, or scroll-effect on hero children (parallax, reveal, fade-in).
  • Any decorative SVG that is not the two-circle .hero__rings.
  • Hardcoded hex colors that do not match the §3 token palette. Replace with token variables.
  • Inline styles on hero markup. All styles must live in hero.css.
  • Any position:fixed or position:sticky on the topbar within the hero.
  • Existing knight asset if it is not a transparent PNG with the proper crop. Replace with /assets/hero/chess-knight.png per §2.

14. Final QA checklist

Master rule: If any element overlaps the headline or reduces readability, the implementation is invalid — regardless of whether individual numeric values match.

Zone integrity (run first)

  • Desktop: no part of the knight, rings, BUILD word, or glow peak (alpha ≥ 0.20) has any pixel left of x = 864px.
  • Desktop: BUILD word's bounding box top edge is at y ≥ 480px (below the lead-in line, which sits around y=408px).
  • Desktop: BUILD word lives entirely in the lower half of the hero (y ≥ 390px).
  • Mobile: no part of the knight, rings, BUILD word, or glow peak has any pixel above y = 360px.
  • Knight is visually secondary to the headline — never the dominant element on the page.
  • Rings sit on the right side only; never overlap text.

Desktop @ 1440px

  • Hero is exactly 780px tall and 1440px wide; no horizontal scroll.
  • Background is flat #F2EFEA; no gradient visible at any corner.
  • Headline reads on exactly two lines: "Understand deeply." / "Build fast." — line breaks come from flex-basis:100% on .w-build, not from <br>, not from natural wrap.
  • Period after "deeply" and after "fast" is rendered as a 14×14 purple square, not a glyph period.
  • "Systems" in the subline is rendered in #6E3CFF with weight 500 and no underline.
  • Lead-in shows a 32px black rule, then "Bring a problem", then a 6×6 purple square.
  • Topbar logo (left), 4 nav links (center-left), "Let's talk" CTA (right) align at the same vertical center, 24px from the top.
  • "Let's talk" CTA is solid black (#0A0A0A), white text, 6px radius, with a 6×6 purple square 6px after the label.
  • Knight image bleeds off the right edge by 60px and sits flush with the bottom of the hero.
  • Outer ring is black 1.25px; inner ring is purple 1px; both stroke-only with no fill.
  • BUILD word reads at 340px, sits behind the knight (z-index 2), and uses rgba(110,60,255,0.10).
  • Purple glow peaks behind the rings (right side) and fades to transparent before the headline.
  • No animation, no transition, no parallax on any hero element.

Mobile @ 390px

  • Hero is 720px tall and 390px wide; no horizontal scroll; no horizontal scrollbar caused by the glow or knight overflow.
  • Topbar shows only the logo (left) and a 3-bar hamburger (right). Nav links and "Let's talk" are removed from the DOM-render at this breakpoint (display:none).
  • Headline reads on exactly four lines: "Understand" / "deeply." / "Build" / "fast." — line breaks come from flex-basis:100% on every .word, not from <br>, not from natural wrap.
  • Headline font-size is 54px; line-height 0.95; the purple square dots after "deeply" and "fast" are 9×9 px.
  • Subline reads "Paid media. Systems. Scale." at 14px.
  • Lead-in shows a 24px black rule, then "Bring a problem" at 12px / weight 500, then a 5×5 purple square (required on mobile, same as desktop).
  • Knight is 280px wide, bleeds off the right edge by 30px, flush with the bottom.
  • BUILD word reads at 130px and sits above the knight's lower half visually but behind it in stack (z-index 2 vs 4).
  • Glow is visible top-right behind the rings; opacity matches desktop.
  • Container is properly clipped — overflow:hidden on .hero hides knight/glow overflow.
